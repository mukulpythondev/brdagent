Project: Secure Pay Wallet - Multi-Modal Requirements Demo

Business Context:
Build a secure digital wallet dashboard for retail customers. The system should consolidate fragmented product notes, compliance guidelines, and UI wireframes into one explainable requirements document.

Functional Requirements:
1. AUTH-101: Users must be able to log in using Google or Microsoft SSO so onboarding is fast for enterprise banking customers.
2. AUTH-102: Security policy requires username and password only. Third-party identity providers, SSO, and federated login must be blocked for every customer account.
3. TXN-201: The dashboard must show account balance updates in real time using socket-based updates.
4. TXN-202: Transaction history must support search, filters, and export for the previous 12 months.
5. NOTIF-301: Send SMS and email alerts for every wallet transaction above INR 5,000.
6. ADMIN-401: Compliance officers must see a read-only audit timeline for login attempts, payment approvals, and profile changes.

Non-Functional Requirements:
1. PERF-501: Balance queries must respond in under 300 milliseconds at peak traffic.
2. SCALE-502: The API must support at least 10,000 concurrent active sessions.
3. SEC-503: All payment and cardholder data must comply with PCI-DSS controls.
4. PRIV-504: Customers must be able to request GDPR-style data deletion, but financial audit records must also be retained for 7 years.

Decision Expectations:
The AI system should extract requirements, detect conflicts, explain the source of each decision, show confidence levels, and route short/simple input to a fast model while using an advanced model for conflict reasoning and final synthesis.
